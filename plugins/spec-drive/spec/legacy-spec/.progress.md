## Learnings

- Ralph-specum v3.1.2 has 8 agents, 5 skills, 11+ commands, hooks, templates, schema — substantial reference architecture
- All four target CLIs (Claude Code, Codex, Gemini, Kiro) use plain Markdown for context loading — artifact format is already compatible
- Think-tank:debate costs ~40K tokens per run — needs to be opt-in, not default for all phases
- Kiro's EARS notation for requirements is interesting but incompatible with existing US/AC format. Skip for now.
- The stop-watcher hook is the linchpin of autonomous execution — must be replicated carefully
- Project-first structure (P###/spec/) requires Antfarm spec_dir changes — recommend as optional, not default
- Constitution phase can be a simple template expansion (load principles.md) — no LLM call needed
- Ralph-specum's multi-directory support via path-resolver.sh already solves the "where do specs live" problem
- Gab's non-negotiable: project-first (`P###-name/spec/`) is THE default, not optional. Multi-directory resolution excluded from scope.
- Gab's non-negotiable: idea.md replaces constitution phase. No separate constitution step needed.
- Gab's non-negotiable: document chain = context flow. No template variables between phases. Agents read full files.
- Gab's non-negotiable: fresh context per task = only task definition + .progress.md. Prevents context window exhaustion.
- Gab's non-negotiable: think-tank:debate is development-time only (stress-test prompts). NOT a runtime feature of the shipped plugin.
- Scoped to 6 agents (researcher, PM, architect, task-planner, executor, QA). Plan-synthesizer and refactor-specialist deferred to v2.
- Antfarm integration explicitly out of scope — plugin is Claude Code first. Antfarm can consume artifacts later.
- State file named `.spec-drive-state.json` to enable A/B comparison with ralph-specum side-by-side.
- Recovery mode (auto-fix task generation) deferred to v2 — too much complexity for initial release.
- Design: 32 files total in plugin (1 plugin.json, 6 agents, 9 commands, 5 skill files, 3 hook files, 6 templates, 1 schema, 1 package.json)
- Stop-watcher needs dual discovery: check cwd first (user in project dir), then scan projectRoot (user elsewhere). Both needed.
- Phase checklists in skill reference file (not hardcoded) — enables customization per AC-10.4
- Parallel execution uses isolated .progress-task-N.md files + flock for tasks.md/git — proven pattern from ralph-specum
- Cascading config: project-level > global > defaults. Keeps plugin generic while allowing per-project overrides.
- Cancel command keeps project files by default, only deletes state. Safer than ralph-specum's full rm -rf.
- Test scripts placed in test/ directory (not scripts/ from tasks.md) per user override. All 4 pass: structure (43 checks), hooks (9 checks), commands (36 checks), schema (8 checks). Total 34 plugin files (>= 32 threshold).
- Agent prompt budget: < 4,000 tokens each (NFR-2). Executor is largest due to autonomous execution constraints.

- Task planning: 4 analysis agents are independent (parallel creation). Executor depends on skills (delegation-principle).
